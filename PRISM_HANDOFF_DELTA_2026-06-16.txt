CRITICAL CORRECTIONS TO THE JUNE 14 HANDOFF
════════════════════════════════════════════════════════════════════

  1. DEPLOY GUARD — NEW MANDATORY STEP
     After `CI=false npm run build`, ALWAYS run:
       node scripts/guard-deploy.js
     THEN run the gh-pages push.

     Full safe recipe:
       CI=false npm run build
       node scripts/guard-deploy.js          ← NEW — aborts if build broken
       cp build/index.html build/404.html
       cat build/CNAME                        ← must print app.winwithone.com
       npx --yes gh-pages -d build --dotfiles -r "https://${PAT}@github.com/presidentsuarez/brokerage.git"
       git add -A && git commit -m "..." && git pull --rebase && git push origin main

     WHY: on June 16 a syntax error caused CRA to fail silently when
     the build output was piped through grep (losing the exit code).
     gh-pages then published an empty build/ dir, unmapping the
     custom domain and serving a 404 for ~2 minutes until the fixed
     build deployed.  guard-deploy.js catches this before it hits
     gh-pages — it verifies index.html, 404.html, CNAME value, and
     the main.*.js bundle all exist.

  2. NEVER PIPE THE BUILD COMMAND
     Bad:  CI=false npm run build 2>&1 | grep ...   ← swallows exit code
     Good: CI=false npm run build
           Then check output separately if needed.
